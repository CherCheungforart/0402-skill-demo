# 转移 React 网站到 Coze 的完整指南

要将你现在的这个自定义 React 交互网站（`ios-interaction-page`）“转移”到 Coze（扣子）上，有几种不同的理解方式。因为 Coze 本质上是一个 AI Bot 构建平台，而你现在拥有的是一个**纯前端网页**。

以下是三种不同的“转移”方式，取决于你想要达到的最终目的：

---

## 方式一：让 Coze Bot 在对话中展示这个网页（作为 Web App 插件/卡片）

**目的**：用户在 Coze 里面和你聊天，聊到一半，Coze 弹出一个卡片，卡片里面内嵌了你这个漂亮 iOS 变装界面。

**操作步骤**：
1. **部署你的前端网页**：
   Coze 无法直接运行你电脑上的代码。你需要先把这个打包好的 `ios-interaction-project.zip` 部署到一个公网服务器上，获得一个真实的网址（例如：`https://my-kpop-app.vercel.app`）。
   *   **最快方法**：使用 [Vercel](https://vercel.com/) 或 [Netlify](https://netlify.com/) 免费部署你的 React 项目。
2. **在 Coze 创建插件 (Plugin)**：
   *   进入 Coze，创建一个新插件。
   *   插件的类型选择为提供一个 URL，填入你部署好的网页链接。
3. **将网页作为卡片返回**：
   *   当用户在 Coze 输入“我要变装”时，配置你的 Bot 触发该插件。
   *   Bot 会在对话流中渲染一个 WebView 卡片，里面运行的就是你部署的这个网页。

**优点**：完美保留你精心设计的所有 UI 交互和动画。
**缺点**：只是把你的网页嵌在 Coze 的对话框里，你的网页依然需要自己部署。

---

## 方式二：放弃当前 UI，将逻辑复刻为 Coze 原生工作流 (Workflow)

**目的**：你不想自己管服务器了，你想把这套“选择功能 -> 选择风格 -> 出方案 -> 出视频”的逻辑，变成 Coze 原生的对话体验。

**操作步骤**：
1. **舍弃 React 代码**：你不需要上传 `Home.tsx` 或任何前端代码，Coze 有自己的一套标准聊天界面。
2. **搭建 Coze 工作流**：
   *   创建一个 Workflow。
   *   添加一个**用户输入节点**，让用户上传图片和输入需求。
   *   添加一个**大模型节点**，将我之前帮你提取的 `SKILL.md`、模板等作为 Prompt 写进去。让它输出那 7 段长文本。
   *   添加一个**画图/视频生成插件**（如火山引擎视频插件），将上一步生成的提示词传给它。
3. **配置多轮对话 (快捷键/选项)**：
   利用 Coze 的“快捷指令”或“选项卡片”功能，复刻“生成图片/视频”、“甜美/潮流风格”的选择体验。

**优点**：真正实现了 AI 变身能力，零代码维护。
**缺点**：**你将完全失去现在这个精心设计的、带有打字机动画和完美 iOS 间距的纯白 UI 界面**。Coze 只能提供它标准的气泡聊天框。

---

## 方式三：用你的 UI 壳，接 Coze 的“大脑”（通过 Coze API）

**目的**：你极其喜欢现在这个 React 界面，不想放弃它，但你想让里面的文字和视频变成真实的 AI 生成的，而不是写死的。

**操作步骤**：
1. **在 Coze 搭建纯逻辑 Bot**：
   按照【方式二】把核心的大模型逻辑、提示词和视频生成插件在 Coze 里配置好，并发布为 API 服务。
2. **修改 React 代码**：
   *   回到我们现在的代码中。
   *   当用户点击选项 A 后，不再触发 `setTimeout` 显示写死的文本。
   *   而是写一段 `fetch` 请求，调用 Coze 提供的 API 接口。
   *   将 Coze API 返回的真实方案和视频 URL 渲染到现在的界面里。
3. **部署**：
   将修改后的前端代码部署到 Vercel 上。

**优点**：完美结合了你设计的 UI 和 Coze 的强大 AI 能力。
**缺点**：需要编写一定的后端请求代码（对接 Coze API），并且依然需要自己部署前端网页。

---

## 总结：你的选择是？

*   如果你想**保留当前的精美 UI**：选择 **方式一**（直接作为链接嵌进去）或 **方式三**（通过 API 深度绑定）。
*   如果你只想要**真实的变装功能**，不在乎界面长什么样：选择 **方式二**（完全在 Coze 里用搭积木的方式重做）。
